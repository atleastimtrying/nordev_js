Concepts we will be covering:

  • namespacing
  • encapsulation
  • singletons
  • pseudo classes
  • events

1: Namespacing

  • namespacing prevents libarary clashes
  • it's a simple step that can tidy your entire codebase
  • it makes your code distict from libraries
  • It can make your code more expressive
  • e.g. window.college.teachers.oldest();

2: Encapsulation

  • Javascript methods are public by default
  • To make a method private contain it within a scope
  • methods in a higher scope can be over written
  • Don't forget the var keyword
  • use closures to seperate your code.
  • one closure per file encourages better structure via window.

3: Singletons

  • in js the singleton pattern can be a simple object literal
  • it helps re enforce your namespacing and group similar functions together
  • it can give you an idea as to the kinds of structures you want to expose to your user
  • A good use of a singleton is for utility functions

4: pseudo class structures

  • you can emulate classes in js
  • use classes for objects with specific behavior
  • dealing with collections is neater with an array of instances
  • an instance can contain refereces to specific bits of the DOM

5: events

  • sometimes referred to as the observer pattern or publish subscribe.
  • It's a pattern built into the browser and drives a great deal of JS behavior
  • it allows for decoupling and discrete bits of code
  • I use jquery's on and trigger methods
  • Events allow for very flexbile designs in your programming
